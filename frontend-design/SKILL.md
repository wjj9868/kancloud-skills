# 前端设计技能

本技能指导创建独特、生产级的前端界面，避免 generic 的"AI 风格"。实现真正可用的代码，注重美学细节和创意选择。

用户提供的需求包括：组件、页面、应用或界面的设计需求，可能包含用途、受众或技术约束的背景信息。

## 设计思维

在编码之前，理解需求并确定大胆的美学方向：
- **目的**：这个界面解决什么问题？谁是目标用户？
- **调性**：选择一个极端风格：极简主义、极繁主义、复古未来主义、自然有机、奢华精致、童趣玩味、编辑杂志风格、粗野主义、装饰艺术、柔和粉彩、工业实用等。灵感来源于此，但要设计出真正符合美学方向的独特方案。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：什么让人过目不忘？最令人记住的一点是什么？

**关键**：选择清晰的概念方向并精确执行。极繁主义和精致的极简主义都可以——关键是意图明确，而非强度。

然后实现可工作的代码（HTML/CSS/JS、React、Vue 等），确保：
- 生产级且功能正常
- 视觉上令人印象深刻且难忘
- 具有清晰美学观点的一致性
- 每个细节都经过精心打磨

## 前端美学指南

### 排版选择

选择美丽、独特、有趣的字体。避免通用字体（如 Arial、Inter），选择能够提升前端美学的独特选择意想不到、有个性的字体组合。用独特的展示字体搭配精致的正文字体。

### 色彩与主题

致力于统一的美学。使用 CSS 变量保证一致性。带有锐利点缀的主色调比胆怯、均匀分布的调色板更出色。

### 动效与动画

使用动画实现效果和微交互。优先使用纯 CSS 解决方案。React 中使用 Motion 库。关注高冲击力时刻：一个精心编排的页面加载配合交错揭示（animation-delay）比分散的微交互更能创造惊喜。使用滚动触发和悬停状态来制造惊喜。

### 空间构图

意想不到的布局。不对称、重叠、对角线流动、打破网格的元素、慷慨的负空间或受控的密度。

### 背景与视觉细节

创造氛围和深度，而非默认纯色。添加符合整体美学的上下文效果和纹理。应用创意形式如渐变网格、噪点纹理、几何图案、层叠透明度、戏剧性阴影、装饰性边框、自定义光标和颗粒叠加。

## 禁止使用的设计

永远不要使用通用的 AI 生成美学：
- 过度的字体族（Inter、Roboto、Arial、系统字体）
- 陈词滥调的调色板（尤其是白底上的紫色渐变）
- 可预测的布局和组件模式
- 缺乏上下文特色的模板化设计

要有创意，做出意想不到的选择，这些选择感觉是真正为上下文设计的。不要千篇一律。在浅色和深色主题、不同的字体、不同的美学之间变化。永远不要在多次生成中趋同于常见选择。

**重要**：实现复杂度要匹配美学愿景。极繁主义设计需要包含大量动画和效果的复杂代码。极简主义或精致的设计需要克制、精确和仔细注意间距、排版和细微细节。优雅来自于很好地执行愿景。
